# Plan: Fork Sync → Issues → Auto-Implement (Kilo CLI + SAP AI Core)

## Goal

Весь код реализуется через **Kilo CLI agent** с провайдером **sap-ai-core** и моделью **anthropic--claude-4.6-opus** в **auto mode**. Полный автоматический pipeline:

1. **Отслеживает новые features** в upstream fork-репозиториях (claude-code, kilocode, cline, aider, opencode)
2. **Создаёт GitHub Issues** с лейблом `auto-implement` на основе обнаруженных feature gaps
3. **Kilo CLI** подхватывает issues каждые 2 часа и реализует их автономно

## Принцип

Каждый workflow использует **одну и ту же формулу** вызова Kilo CLI:

```bash
kilo run "$PROMPT" --auto -m "sap-ai-core/anthropic--claude-4.6-opus"
```

- `--auto` — автоматическое одобрение всех tool permissions (файловые операции, bash, etc.)
- `-m sap-ai-core/anthropic--claude-4.6-opus` — модель через SAP AI Core
- `AICORE_SERVICE_KEY` + `AICORE_RESOURCE_GROUP` — аутентификация через env secrets

## Изменения

### 1. Исправить `sync-to-issues.yml`

**Файл**: `.github/workflows/sync-to-issues.yml`

**Изменения**:
- Вместо `analyzer.analyzeChanges([])` → запускать `SyncManager.sync()` для получения актуальных feature gaps
- Добавить `schedule: cron '0 7 * * *'` как fallback
- Фильтрация: только категории `tool`, `provider`, `command`, `mcp`, `hook`, `skill`, `config`
- Исключить: `ui`, `security`, `performance`
- Увеличить лимит до 10 issues/запуск
- Сортировка: small effort first (быстрые wins)
- Лимит: максимум 3 issues с `estimatedEffort: 'large'`

**Ключевой diff Node.js скрипта**:
```diff
- // Re-run analysis to get structured data
- const analyzer = new FeatureAnalyzer();
- const analysis = analyzer.analyzeChanges([]);
+ // Run sync to get real changes from upstream repos
+ const { SyncManager } = await import('./dist/sync/index.js');
+ const syncManager = new SyncManager({ githubToken: process.env.GH_TOKEN });
+ const report = await syncManager.sync({ force: false });
+ const analyzer = new FeatureAnalyzer();
+ const analysis = analyzer.analyzeChanges(report.changes);
```

**Добавить whitelist категорий**:
```javascript
const ALLOWED_CATEGORIES = ['tool', 'provider', 'command', 'mcp', 'hook', 'skill', 'config'];

const eligibleGaps = analysis.featureGaps
  .filter(gap => {
    const gapPriority = priorityOrder[gap.priority] || 0;
    return gapPriority >= minPriorityValue 
      && gap.missingInOur 
      && ALLOWED_CATEGORIES.includes(gap.category);
  })
  .sort((a, b) => {
    // Small effort first, then by priority
    const effortOrder = { small: 3, medium: 2, large: 1 };
    const effortDiff = (effortOrder[b.estimatedEffort] || 0) - (effortOrder[a.estimatedEffort] || 0);
    if (effortDiff !== 0) return effortDiff;
    return (priorityOrder[b.priority] || 0) - (priorityOrder[a.priority] || 0);
  });

// Limit large-effort issues to 3
let largeCount = 0;
const finalGaps = eligibleGaps.filter(gap => {
  if (gap.estimatedEffort === 'large') {
    largeCount++;
    return largeCount <= 3;
  }
  return true;
});
```

## Итоговый pipeline

```
┌─────────────────────────────────────────────────────────────┐
│  DAILY 6:00 UTC — repo-sync.yml                             │
│  Fetches commits from 5 upstream repos (claude-code,        │
│  kilocode, cline, aider, opencode), classifies changes      │
│  by category, uploads sync-report.md artifact               │
└──────────────────────┬──────────────────────────────────────┘
                       │ workflow_run (success) OR schedule 7:00
                       ▼
┌─────────────────────────────────────────────────────────────┐
│  DAILY 7:00 UTC — sync-to-issues.yml                        │
│  Re-runs SyncManager.sync(), analyzes feature gaps,         │
│  filters by category whitelist + priority, deduplicates,    │
│  creates max 10 GitHub Issues with 'auto-implement' label   │
└──────────────────────┬──────────────────────────────────────┘
                       │ issues with 'auto-implement' label
                       ▼
┌─────────────────────────────────────────────────────────────┐
│  EVERY 2 HOURS — auto-implement.yml                         │
│  Picks oldest issue → creates branch → runs:                │
│  kilo run "$PROMPT" --auto -m sap-ai-core/anthropic--       │
│  claude-4.6-opus → verifies (typecheck/lint/test/build)     │
│  → creates PR → updates issue                              │
└──────────────────────┬──────────────────────────────────────┘
                       │ if CI fails
                       ▼
┌─────────────────────────────────────────────────────────────┐
│  ON FAILURE — ci-auto-fix.yml                               │
│  Collects logs → lint:fix + format → if not enough:         │
│  kilo run "$PROMPT" --auto -m sap-ai-core/anthropic--       │
│  claude-4.6-opus → commits fix → pushes                    │
└──────────────────────┬──────────────────────────────────────┘
                       │ all green
                       ▼
                  Human Review → Merge
```
