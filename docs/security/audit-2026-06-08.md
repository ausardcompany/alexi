### W3 - `update-homebrew.yml` force-moves release tags from automation

- File: `.github/workflows/update-homebrew.yml:48-62`.
- Trigger: `pull_request_target [closed]` on branches starting with
  `auto/sync-upstream-` or `release/`. Guarded by
  `merged == true` (correct).
- Risk: the workflow runs `git tag -f "$TAG" "$SHA"` then
  `git push origin ":refs/tags/$TAG"` followed by `git push origin
  "$TAG"`. A force-moved tag changes what consumers download from a
  pinned version with no audit trail in the tag itself. Combined
  with auto-merge of sync-upstream PRs, an upstream-injected change
  could shift the meaning of a published `vX.Y.Z` tag.
- Recommendation: convert "tag exists at different SHA" into a hard
  failure (`exit 1`) with a Slack/issue alert instead of silently
  force-pushing. Releases that need a re-tag should be a manual
  `workflow_dispatch` action, not part of the auto path.

### I2 - `AllowEverythingPermission` class is unused and uses unsafe glob-to-regex

- File: `src/permission/allow-everything.ts`.
- Not imported anywhere in `src/` or `tests/`. Currently dead code.
- Even unused, the implementation has two latent issues that should
  be fixed if/when the class is wired in:
  1. `matchesPattern` (line 53) builds an unanchored regex by only
     replacing `*` with `.*`. Pattern `foo` matches anywhere in the
     resource path, and metacharacters (`.`, `(`, `[`, etc.) are
     not escaped. A pattern like `..` would match every two-char
     substring rather than a literal `..`.
  2. The default `enabled: false` returns `decision: 'ask'`, but
     once enabled there is no per-action filtering - any tool with
     any action is granted regardless of `request.action`. The
     class name is honest, but it should require explicit opt-in
     via env var (e.g. `ALEXI_ALLOW_EVERYTHING=1` plus a TTY
     interactive confirm) before `setEnabled(true)` takes effect.
- Recommendation: either delete the file, or guard `setEnabled` and
  fix the regex. Open as `chore(core)` follow-up; not blocking.
