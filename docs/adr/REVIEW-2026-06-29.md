## Summary

Fourth architecture run. Two weeks of activity since `REVIEW-2026-06-15`:
roughly nine more `feat(sync): apply upstream changes` PRs, a handful of
genuine feature commits (`feat(core): auto-disable routes...`,
`feat(providers): plumb topK through SAP orchestration provider`,
`feat(providers): add enforceStrictSchema for OpenAI Responses strict-mode`,
`feat(cli): surface per-routed-model usage in /cost today`,
`fix(tools): bound webfetch response body`), and a Dependabot wave.

**Headline architectural news in this run:**

- **Three brand-new top-level `src/` directories** -- `src/router/`,
  `src/session/`, and `src/ui/` -- were merged via the upstream sync
  stream on 2026-06-17 (#798) and 2026-06-19 (#806). Each contains
  exactly one file, each file is exactly one comment line, no exports,
  no consumers. They are the exact same anti-pattern as F12 from the
  previous review, repeated three times. They are also direct
  violations of ADR 001 -- "Adding a new top-level directory under
  `src/`" is on the explicit list of changes that require an ADR
  before merge, and no ADR was filed. This is **F13** below.
- **F3 is now half-fixed.** `src/providers/index.ts` re-exports
  `type CompletionResult` (line 35) and `type TokenUsage` (line 37).
  The callers (`src/core/agenticChat.ts:19`,
  `src/core/streamingOrchestrator.ts:16`) have NOT been updated to use
  the new re-exports, so the layering violation persists, but the
  provider-surface side of the fix has landed silently. Likely
  introduced by `feat(providers): plumb topK...` or
  `feat(providers): add enforceStrictSchema...`. Updating the two
  callers is now a one-line diff each.
- **F10 / F11 / F12 are still present, unchanged.** Two weeks have
  passed since they were called out as `block`; `src/core/billing.ts`,
  `src/core/catalog.ts`, and `src/core/integration/schema.ts` are all
  still on disk. Recommendation 1 from REVIEW-2026-06-15 (a single
  cleanup PR) was not executed.
- **F5 is still present, unchanged.** `src/permission/PermissionView.kt`
  and `PermissionViewTest.kt` have now been flagged three reviews in a
  row.
- **F6 is unchanged.** Five SQL files in `src/core/migration/`, no
  Prisma / SQLite dependency. No new migration files appeared this
  cycle, but no resolution either.

The cumulative picture is that the architecture verticals are
discovering problems faster than the engineering vertical is closing
them. Every F10/F11/F12-style block has now had a partner finding
arrive in the next sync cycle (F13 is essentially "F12 happened three
more times"). The sync allowlist mitigation proposed in
Recommendation 3 of REVIEW-2026-06-15 is the highest-leverage move
this repo can make right now.

**No new ADR in this run.** ADR 001 already declares "new top-level
`src/` directory" as ADR-required, so F13 is a violation of an
existing rule, not a new rule category. The right output is the
cleanup PR plus the long-overdue ADR 002 baseline.

### F13 -- `block` -- THREE new empty top-level dirs added by sync (NEW)

The 2026-06-17 and 2026-06-19 upstream-sync PRs added three brand-new
top-level directories under `src/`. Each contains exactly one file,
each file is a single comment line, none has any exports or consumers:

```sh
$ ls src/router/ src/session/ src/ui/
src/router/:
index.ts

src/session/:
data.ts

src/ui/:
timers.ts

$ wc -l src/router/index.ts src/session/data.ts src/ui/timers.ts
1 src/router/index.ts
1 src/session/data.ts
1 src/ui/timers.ts

$ cat src/router/index.ts
// Add new router enhancements based on upstream

$ cat src/session/data.ts
// Integrate new session data model changes

$ cat src/ui/timers.ts
// Static timer logic for enhanced testing
```

Verification that none of them is wired anywhere in the codebase:

```sh
$ grep -rEn "from ['\"]\\.\\.?/(\\.\\./)*router/" src/ --include='*.ts' --include='*.tsx'
# (no output)
$ grep -rEn "from ['\"]\\.\\.?/(\\.\\./)*session/data" src/ --include='*.ts' --include='*.tsx'
# (no output)
$ grep -rEn "from ['\"]\\.\\.?/(\\.\\./)*ui/timers" src/ --include='*.ts' --include='*.tsx'
# (no output)
```

This is severity `block` for three compounding reasons:

1. **ADR 001 explicit rule violation.** ADR 001, decision point 3,
   first bullet: *"Adding a new top-level directory under `src/`"*
   requires an ADR before merge. Three top-level directories were
   merged with no ADRs (`docs/adr/` currently contains only
   `001-establish-adr-process.md` plus four review files).
2. **Name collision with established concepts.** A top-level
   `src/router/index.ts` next to `src/core/router.ts` is exactly the
   kind of two-routers-same-name confusion ADR 001 was written to
   prevent. A top-level `src/session/` next to
   `src/core/sessionManager.ts`, `sessionBusy.ts`, `sessionClose.ts`,
   `sessionContext.ts` is the same trap. A top-level `src/ui/`
   alongside the existing `src/cli/tui/` is the same trap a third
   time. Once anything actually depends on these dirs, the import
   site will become ambiguous between `core/router` and `router/`
   for any developer skimming the diff.
3. **Foreign-code smell.** The comments use phrasing that does not
   match Alexi's own (`"based on upstream"`, `"new session data
   model"`, `"for enhanced testing"` for a *timers* file with no
   tests anywhere). This is the same pattern as F10/F11/F12 -- code
   imported wholesale from a different repository without filtering.

Resolution: the cleanup PR proposed in Recommendation 1 should also
`git rm -r src/router/ src/session/ src/ui/`. If any of those names
will be used for real future work, that future work files its own ADR
and creates the directory then.

## Recommendations

These are deliberately the same recommendations as REVIEW-2026-06-15
plus the F13 addition, with priorities re-ordered to reflect what has
become more urgent.

1. **Combined cleanup PR -- `chore(sync)` -- delete six orphan paths.**
   In one PR, run:

   ```sh
   git rm src/core/billing.ts                  # F10
   git rm src/core/catalog.ts                  # F11
   git rm -r src/core/integration              # F12
   git rm -r src/router src/session src/ui     # F13
   ```

   Then verify with `npm run typecheck && npm run lint && npm run build`.
   Six files / four directories, zero non-test references in the
   codebase, expected diff is purely deletions. This collapses F10,
   F11, F12, and F13 into a single small reviewable change.

2. **Sync-allowlist guardrail (`role-infrastructure`).** Add to the
   sync workflow a step that diffs the synced tree against an explicit
   set of expected paths and **fails the run** if it sees any of:

   - New top-level `src/<name>/` dirs not present before AND not
     accompanied by `docs/adr/NNN-*.md` in the same diff.
   - Files matching `payment*`, `billing*`, `*credentials*`,
     `*stripe*`.
   - Empty-on-purpose files (size < 200 bytes containing only a
     comment) outside `docs/`.

   This is the single highest-leverage mitigation in the current
   review; without it F10..F13 will keep recurring.

3. **Finish the F3 / F3b migration (`role-engineering`).** The
   provider re-exports already exist. Open one PR scoped
   `refactor(core)` that:

   - Adds `buildSessionHeaders` to `src/providers/index.ts`'s public
     exports (matching the pattern already used for
     `CompletionResult` and `TokenUsage`).
   - Adds a `listDeployments()` helper to `src/providers/index.ts`
     wrapping `DeploymentApi.deploymentQuery(...)`.
   - Updates `src/core/agenticChat.ts:19`,
     `src/core/streamingOrchestrator.ts:16`, and
     `src/cli/commands/models.ts:6` to import only from
     `'../providers/index.js'` (and removes the direct
     `@sap-ai-sdk/ai-api` import in models.ts).

   This closes F2, F3, and F3b in one motion. The diff is mostly
   import-path rewrites.

4. **Delete the Kotlin files (F5).** Third review flagging it; a
   single-PR `git rm src/permission/PermissionView.kt
   src/permission/PermissionViewTest.kt` scoped `chore(permission)`
   resolves it permanently.

5. **F6 -- migration directory decision.** Same recommendation as
   REVIEW-2026-06-15: either ADR + `better-sqlite3` dep + loader +
   tests, or `git rm -r src/core/migration/`. Still the worst of
   both worlds today.

6. **F7 -- update role-architecture.md module map.** Replace the
   inline list with a one-line pointer to `docs/ARCHITECTURE.md`. The
   inline list has been stale for three consecutive reviews.

7. **ADR 002 -- baseline.** File the bulk-backfill ADR once
   Recommendations 1 and 4 are merged. ADR 002 should:

   - Memorialise the post-cleanup `src/` top-level set (32 dirs,
     after F13 removals) as the accepted baseline, with one-sentence
     justifications.
   - Formalise the `__tests__/` and `tests/` cross-layer exception
     (resolves F9).
   - Reference ADR 001's trigger list and clarify that "moved here
     from upstream" is not a sufficient ADR justification on its own.
